Capabilities

Everything you need to debug

A complete network inspection layer. Drop it in, attach the interceptor, and every HTTP call is captured and viewable instantly.

OkHttp Interceptor

Captures every HTTP request and response automatically. No manual instrumentation required.

Detailed Timing

DNS lookup, TCP handshake, TLS negotiation, TTFB, and body transfer — broken down per request.

In-App Console

A searchable, filterable Jetpack Compose UI that renders the traffic log directly inside your app.

Request Inspector

Headers, request body, response body with JSON pretty-print, and a visual timing chart — all in one place.

Sensitive Data Redaction

Configurable redaction of headers, query parameters, and JSON fields before any value is stored.

Session Management

Traffic grouped by app session with device metadata — making cross-session debugging straightforward.

Export & Share

Export logs as JSON, Plain Text, Markdown, or HTML and share via the Android share sheet in one tap.

cURL Commands

Generates a ready-to-run cURL command from any captured request for instant local reproduction.

Persistent Storage

Room database backend with automatic retention policies and configurable size caps.

Material 3 Dark Theme

Purpose-built dark UI using Material Design 3. Feels native in your development workflow without distracting from your app.

Interface

Built for developers

A polished, non-intrusive console that stays out of your way — until you need it.

Integration

Up and running in minutes

Four steps to add Kulse to any Android project using OkHttp or Retrofit.

01

Add the dependencies

Add both modules to your app-level build.gradle.kts. You can depend on kulse-core alone if you plan to build a custom UI.

build.gradle.kts
dependencies { implementation("com.iosdevc.android:kulse-core:<version>") implementation("com.iosdevc.android:kulse-ui:<version>") }
02

Initialize in your Application class

Call Kulse.install() once at app startup. Pass a KulseConfig to customize behaviour, or omit it to use the defaults.

Application.kt
class MyApp : Application() { override fun onCreate() { super.onCreate() Kulse.install( context = this, config = KulseConfig( sensitiveHeaders = setOf("Authorization", "Cookie"), excludedHosts = setOf("analytics.example.com"), ), ) } }
03

Attach the interceptor to OkHttp

Add both the interceptor and the event listener factory to capture full timing data. Works with Retrofit out of the box.

NetworkModule.kt
val client = OkHttpClient.Builder() .addInterceptor(Kulse.interceptor()) .eventListenerFactory(Kulse.eventListenerFactory()) .build() // Works seamlessly with Retrofit val retrofit = Retrofit.Builder() .baseUrl("https://api.example.com/") .client(client) .build()
04

Open the console

Launch the standalone activity from anywhere in your app, or embed the KulseConsole composable inside your own screen.

Kotlin
// As a standalone Activity KulseActivity.start(context) // Or embed the Composable in your own screen KulseConsole()
Reference

Configuration

Fine-tune storage limits, privacy controls, host filtering, and log verbosity with KulseConfig.

KulseConfig.kt
KulseConfig( // ── Storage ──────────────────────────────────────────── maxStorageSize = 256L * 1024 * 1024, // 256 MB maxAge = 14.days, // auto-purge after 14 days // ── Body size limits ─────────────────────────────────── maxRequestBodySize = 1L * 1024 * 1024, // 1 MB maxResponseBodySize = 8L * 1024 * 1024, // 8 MB // ── Privacy / redaction ──────────────────────────────── sensitiveHeaders = setOf("Authorization", "Cookie", "Set-Cookie"), sensitiveQueryParams = setOf("api_key", "token"), sensitiveJsonFields = setOf("password", "ssn"), // ── Host filtering ───────────────────────────────────── excludedHosts = setOf("crashlytics.googleapis.com"), includedHosts = emptySet(), // if non-empty, only these hosts are logged // ── Control ──────────────────────────────────────────── isEnabled = true, logLevel = LogLevel.DEBUG, )
